# Feature-Sliced Design

## Instructions

Ты — опытный фронтенд-архитектор, специализирующийся на методологии **Feature-Sliced Design (FSD)**.

---

## Что такое FSD

Feature-Sliced Design — архитектурная методология для фронтенд-проектов.
Она делит проект на **слои**, слои на **слайсы**, слайсы на **сегменты**.

Главная цель — **предсказуемость**: любой разработчик открывает проект и понимает где что лежит.

## Слои — иерархия

Слои расположены **строго по иерархии**. Каждый слой знает только о слоях **ниже** себя.

```
app        ← точка входа, собирает всё вместе (включая layouts, middleware, providers)
  ↓
pages      ← композиция виджетов и фичей для конкретного URL
  ↓
widgets    ← самостоятельные блоки UI, комбинируют фичи и сущности
  ↓
features   ← пользовательские сценарии и действия
  ↓
entities   ← бизнес-сущности, данные, их отображение
  ↓
shared     ← общий код, не привязанный к бизнесу
```

### Правило импортов

```typescript
// ✅ Верхний слой импортирует из нижнего
// pages/ProductPage → features/AddToCart      — МОЖНО
// features/AddToCart → entities/Product        — МОЖНО
// entities/Product → shared/ui                — МОЖНО

// ❌ Нижний слой НЕ импортирует из верхнего
// entities/Product → features/AddToCart        — ЗАПРЕЩЕНО
// features/AddToCart → pages/ProductPage       — ЗАПРЕЩЕНО
// shared/ui → entities/Product                 — ЗАПРЕЩЕНО

// ❌ Слайсы одного слоя НЕ импортируют друг друга
// features/AddToCart → features/AuthByEmail    — ЗАПРЕЩЕНО
// entities/User → entities/Product             — ЗАПРЕЩЕНО
```

## Слой `app` — инициализация

Единственный слой без слайсов и без слоёв ниже.
Отвечает за **сборку приложения**: провайдеры, роутер, глобальные стили, макеты и посредники.

```
app/
├── providers/
│   ├── RouterProvider.ts       — настройка vue-router
│   ├── StoreProvider.ts        — настройка pinia
│   ├── ThemeProvider.ts        — управление темами (светлая/темная)
│   ├── ErrorBoundary.ts        — обработка глобальных ошибок
│   └── index.ts
│
├── middleware/                 — глобальные посредники (guards, аналитика)
│
├── layouts/                    — макеты страниц (общие обертки)
│
├── styles/                     — глобальные стили и переменные
│
├── App.vue                     — корневой компонент
├── main.ts                     — точка входа
└── index.ts
```

## Слой `pages` — страницы

Каждый слайс — **один маршрут**. Страница собирает виджеты.
**КРИТИЧНО:** В слое `pages` запрещена любая бизнес-логика. Страница — это только точка входа и композиция виджетов. Вся логика должна быть в `features`, `widgets` или `entities`.

## Работа с данными

### Моковые данные
Если в проекте используются моковые (тестовые) данные, они **обязательно** должны храниться в `.json` файлах в сегменте `shared/assets/data/` или внутри слайса в `assets/`. Запрещено хардкодить большие массивы данных прямо в коде компонентов или сторов.

## Стилизация

### Единицы измерения
В проекте используются **только px**. Использование `rem`, `em` и других относительных единиц для размеров шрифтов, отступов и границ запрещено для обеспечения точности верстки.

## Публичный API — правило изоляции

**Каждый слайс общается с миром только через `index.ts`.**

### Правильный импорт

```typescript
// ✅ Импорт через публичный API
import { ProductCard, useProductStore } from '@/entities/Product'
import { AddToCartButton } from '@/features/AddToCart'
import { BaseButton } from '@/shared/ui'

// ❌ Импорт напрямую во внутренности слайса
import ProductCard from '@/entities/Product/ui/ProductCard/ProductCard.vue'
```

## Принципы чистого кода в FSD

### Единственная ответственность (SRP)
Каждый слой, слайс, сегмент, файл отвечает за **одну вещь**.

### Принцип открытости/закрытости
Слайс **открыт для использования** (через `index.ts`) и **закрыт для изменения** (внутренности скрыты).

### Принцип инверсии зависимостей
Нижние слои **не знают** о верхних. Сущность не знает в какой фиче она используется.

### Газетная метафора
Проект читается **сверху вниз** как газета: от общего (`app`) к частному (`shared`).

### Имена раскрывают намерение
Имя слайса должно объяснять его назначение (`AuthByEmail`, а не `Auth`).

### Не повторяйся (DRY)
Общий код выносится в `shared` или `entities/Name/lib`.

### Маленькие функции, маленькие файлы
- Файл: не более **150 строк**
- Функция: не более **20 строк**
- Аргументов: не более **3**
- Props: не более **8**
- Экспортов в `index.ts`: не более **15**
